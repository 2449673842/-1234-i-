# SciFigure IFC V3.2 升级内容

> 用给下一个 AI 看，快速理解当前项目状态和 v3.2 做了什么。

---

## 一、定位

V3.1 完成了语义感知编辑层（Semantic Scanner / Binding Engine / CodePatch / GroupPanel / LayerTree / AxisGridSpine / Legend / Text Editing）。

V3.2 在 V3.1 基础上新增三条主线——Project Layer、Selection Layer、Text Drag Layer——把单图编辑器升级为项目级科研绘图工作台。

不替换 V3.1 架构，只在之上叠加新的 Layer。

---

## 二、架构分层（V3.2 后的全局视角）

```
Source Layer
  Python Script / CodePatch

Project Layer         ← V3.2A
  Project / Files / Figures / Exports

Figure Layer          ← V3.2A
  FigureId / Manifest / EditLog / Revision

Artist Layer
  GID / Props / Groups / Bindings

Selection Layer       ← V3.2B
  selectedGids / Marquee / Batch Edit / History

Render Layer
  Replay Render / Export
```

## 三、V3.2A — Project Layer（已交付）

### 数据库扩展（`db.ts`）
- `projects` 表：新增 `script`、`file_count` 列
- 新增 `project_files` 表：`id, project_id(FK), original_name, stored_path, columns(JSON), row_count, uploaded_at`
- 新增 `project_figures` 表：`id, project_id(FK), figure_index, session_id, revision`
- 所有外键带 `ON DELETE CASCADE`

### 多文件上传（`server.ts`）
- Multer 磁盘存储到 `data/projects/:id/files/`
- `POST /api/projects/:id/files`：上传 CSV/TSV/XLSX，解析列名和行数
- `GET /api/projects/:id/files`：列出数据集
- `DELETE /api/projects/:id/files/:id`：删除数据库记录 + 物理文件

### 多 Figure 捕获（`renderer/introspector.py`）
- `Figure.__init__` monkey-patch：所有 Figure 实例注册到 `_figure_registry`
- 双重捕获机制（hook + `plt.get_fignums()` 回退）
- 每 Figure 独立 Manifest / EditLog / Revision / SVG

### 文件读取沙箱
- `os.chdir` 切换 CWD 到项目文件目录
- `pd.read_csv` / `pd.read_excel` 包装：`os.path.realpath` → 校验在 `data/projects/:id/files/` 内
- 注入 `_uploaded_file_paths` 字典（文件名 → 实际路径）
- 阻止 `plt.show()` / `plt.savefig()` / URL 协议路径

### 项目 CRUD API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects` | 列表 |
| GET | `/api/projects/:id` | 详情（含 datasets + figures） |
| POST | `/api/projects` | 创建 |
| PUT | `/api/projects/:id` | 更新 |
| DELETE | `/api/projects/:id` | 删除（级联清理） |
| POST | `/api/projects/:id/figures/render` | 多图渲染（带 editLog 重放） |
| GET | `/api/projects/:id/figures` | Figure 元数据 |
| POST | `/api/projects/:id/export` | 导出（单图/全图） |

### AST 安全门
- `renderer/ast_validator.py`：检测禁止模块（os/sys/subprocess/builtins/shutil/socket/urllib/requests）和禁止调用（`__import__`/`eval`/`exec`/`open`/`compile`/`globals`/`locals`）
- 所有 Python 执行路径都经过 AST 校验

### 项目模式 UI
- MainWorkspace Figure Tabs（多 Figure 独立编辑）
- LeftSidebar 项目数据管理面板（上传/删除数据集）
- `project_create` 视图（3 步：上传数据 → 准备脚本 → 创建并进入编辑器）
- `projects` 视图（项目列表/搜索/重命名/删除）

## 四、V3.2B — Selection Layer（已交付）

### 多选
- `selectedGids: string[]` 状态，与旧的 `selectedObject` 反向兼容
- LeftSidebar 图层树：Ctrl/Shift 多选
- RightSidebar：多选时显示批量编辑面板（只显示兼容属性）

### 框选（Marquee）
- SVG 覆层 `<div>` 绘制选择框
- mousedown → mousemove → mouseup 计算矩形区域
- 匹配 `data-scifigure-id` 元素的位置（getBoundingClientRect）
- 设置 `selectedGids` 触发同步

### 批量编辑
- `supportsBatchProp(kind, prop)` 按 artist 种类过滤属性
- 批量编辑写入 editLog 时，逐对象跳过不支持的属性
- 属性控件复用单选面板的渲染逻辑

### 撤销/重做
- 每 Figure 独立 `projectHistory: { past: EditEntry[][], future: EditEntry[][] }`
- undo：弹出 past 最后一个 EditLog，用当前 editLog 推入 future，恢复 past 到 editor
- redo：弹出 future 最后一个 EditLog 推入 past
- 渲染响应写回最新的 manifest/editLog/revision

## 五、V3.2C — Text Drag Layer（已交付）

### Python 端
- `introspector.py`：新增位置 setter（`_PROP_TO_SETTER` 中 `position` → setter）
- `_read_text_props` 输出：x, y, coord_system, ha, va, rotation
- `_APPLY_SINGLE`：支持 position setter

### 前端坐标映射
- 优先从 spine bbox 映射 SVG delta → axes 坐标
- Figure fallback（无 spine 时）
- `grab` 光标指示可拖拽

### 拖拽交互
- mousedown 开始拖拽，mousemove 更新 CSS transform（translate）预览，60fps
- mouseup 计算最终 delta，写入 editLog
- `CSS.escape()` 安全转义 GID 选择器

## 六、Script First 流程

V3.2 默认入口改为 **先脚本后数据**（原本是 Data First：先导入数据 → 选图表类型 → AI 转义 → 编辑器）：

```text
新流程（Script First）     旧流程（Data First）
新建项目                    导入数据
  ├─ 脚本编辑 / .py 上传        ├─ CSV/XLSX → 选择列
  ├─ 数据文件上传                ├─ 图表类型选择
  ├─ AI 提示词复制 + 粘贴        ├─ AI 转义脚本
  └─ 创建 + 渲染进编辑器          └─ 进入编辑器
```

`ProjectCreatePage.tsx` 实现为 3 步骤向导：
1. 上传数据文件（拖拽/选择，本地暂存未提交）
2. 准备脚本（编辑器 + .py 上传 + AI 提示词复制 + AI 结果粘贴区）
3. 创建项目并进入编辑器（统一提交：创建项目 → 批量上传数据 → 保存脚本 → 进入编辑器）

## 七、稳定性修复

### 白屏修复
- `buildReproduciblePython()`：所有 spec 子对象（`figure`/`axes`/`legend`/`font`）空安全默认值 `|| DFLT_*`
- `handleLoadProject` / `handleProjectRender`：try-catch 包裹脚本生成
- `projectIsRendering` 状态：加载/渲染时前端显示 spinner
- 非 custom 项目自动用 `buildReproduciblePython` 生成脚本

### Error Boundary
- `EditorErrorBoundary`（class component）：包裹编辑器区域
- 崩溃后显示错误信息 + "查看 F12 控制台" 提示，不影响导航栏

### 持久化
- `handleNavigate` 同步写入 `sessionStorage`（不是走 effect），刷新后导航状态准确

### Figure 指纹/漂移检测（P1.5）
- `hashlib.sha256`（替代不稳定的 `hash()`）用于 fingerprint
- 服务端对比新旧 gid 集合 → `driftedGids: { missing[], new[] }`
- 前端确认流程

## 八、关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| Figure 捕获 | `__init__` hook + `get_fignums()` 双重 | 脚本内可能 `plt.close(fig)` |
| 路径沙箱 | `os.path.realpath` + `commonpath` | 兼容相对路径和 `_uploaded_file_paths` |
| 框选目标 | 遍历 `validGids` 而非 `selectedGids` | 框选应选中新对象 |
| 坐标映射 | spine bbox → axes coords，figure 回退 | 不依赖 data 坐标（未映射时更稳定） |
| 编辑历史 | 每 Figure 独立 `{past,future}` EditEntry[][] | 避免跨 Figure 污染 |
| 非 custom 渲染 | 始终 `buildReproduciblePython` 忽略 `custom_script` | 保证旧项目兼容 |
| SVG 确定性 | `hashsalt="scifigure-v1"` + `metadata.Date=None` | 减少不必要漂移 |
| Python 幂等性 | `hash()` 换 `hashlib.sha256` | `hash()` 每次进程重启后不同 |
